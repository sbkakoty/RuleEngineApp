Architectural Pattern
The application follows MVVM.
Layers
UI Layer
* SwiftUI Views
* ViewModels
* Navigation
Domain Layer
* Use Cases
* Business Rules
* Validation Logic
* Repository Protocol
Data Layer
* Firebase Services
* Local Storage
* Models

Dependency Injection
Dependencies are injected through initializers.
Example:
RuleListViewModel(
    repository: RuleServiceProtocol
)
